# KRB Vault — AI Context

*Extends `Common/_Meta/VAULT_CONTEXT_BASE.md`. All universal rules apply. This file adds KRB-specific file types, tag hierarchies, and whitewater content conventions.*

---

## 1. Vault Identity

**Public whitewater kayaking community resource.** Safe for any LLM — no personal or confidential information.

- **Maintainer:** Jose Luis Pino (hobby kayaker, not professional affiliation)
- **Audience:** Whitewater kayakers and paddlers using Garmin Instinct wearables
- **Mission:** Bridge the gap between river navigation and hands-free wrist-based GPS tools
- **Repository:** Public GitHub (`Kern-River-Boaters/gpx`)
- **Content scope:** GPS waypoint collections, Obsidian Leaflet maps, river guides, safety notes

**Key constraint:** This vault is openly published. Never reference personal vaults, work systems, or private data. All content must be appropriate for public redistribution.

## 3. File Types & Required Structure

### `type/river_guide` — River guide with embedded map (most common)

Auto-generated by `refresh_gpx_maps.py`. User notes go above the auto-generated block.

**Required frontmatter:**

```yaml
---
tags:
  - type/river_guide
  - river/[river]             # e.g. river/kern/lower
  - feature/[feature]         # e.g. feature/rapid
  - difficulty/[class]        # e.g. difficulty/class_ii_iii  (omit if no rapids)
  - region/[region]           # e.g. region/california
  - season/[season]           # e.g. season/year_round
  - status/[status]           # e.g. status/verified
---
```

**Extended frontmatter** (for waypoint collections):
```yaml
river_section: "Lower Kern"
difficulty_range: "Class II-III"
waypoint_count: 42
last_verified: YYYY-MM-DD
```

**Required sections:**
- User notes / description (above auto-generated block) — optional but encouraged
- `<!-- BEGIN AUTO-GENERATED MAP -->` block — regenerated by script, do not hand-edit

## 4. Tag Hierarchy

**Schema location:** `_Meta/Schemas/obsidian_tag_refactor.json`

### `type/` — Content category (REQUIRED on every file)
- `type/river_guide` — Markdown files with embedded maps and river notes
- `type/gpx` — Raw GPX source files
- `type/data` — GeoJSON data files
- `type/map` — Legacy map-only files (prefer `type/river_guide` for new content)
- `type/tutorial` — Setup guides and how-tos
- `type/script` — Python scripts
- `type/documentation` — README, CONTRIBUTING, SOP files
- `type/meta` — Vault governance files

### `river/` — River system
- `river/kern` — Kern River (general)
- `river/kern/upper` — Upper Kern (Forks to Johnsondale)
- `river/kern/lower` — Lower Kern (Kernville to Lake Isabella)
- `river/kern/north_fork` — North Fork of Kern
- `river/san_joaquin` — San Joaquin River
- `river/rogue` — Rogue River (Oregon)

**Rule:** Use the most specific sub-hierarchy available.

### `difficulty/` — Whitewater classification (use on rapids content)
- `difficulty/class_i` — Beginner (moving water, easy)
- `difficulty/class_ii` — Novice
- `difficulty/class_iii` — Intermediate
- `difficulty/class_iv` — Advanced
- `difficulty/class_v` — Expert (potentially life-threatening)
- Range tags: `difficulty/class_ii_iii`, `difficulty/class_iii_iv`, `difficulty/class_iv_v`

**Rule:** For content covering a range of rapids, use the combined range tag. Omit on non-rapid content (campgrounds, parking).

### `feature/` — What the waypoints represent
- `feature/rapid` — Rapids and drops
- `feature/campground` — Camping areas
- `feature/access` — Put-ins and take-outs
- `feature/parking` — Parking areas and trailheads
- `feature/hazard` — Portages, sieves, keeper holes, dangerous features
- `feature/poi` — Points of interest (waterfalls, hot springs, landmarks)
- `feature/milestone` — River miles, gauges, confluences

**Rule:** A file may have multiple `feature/` tags if it covers mixed waypoint types.

### `season/` — When the river runs
- `season/spring_runoff` — Spring snowmelt (high flows, often dangerous)
- `season/summer` — Summer low-flow conditions
- `season/fall` — Fall flows
- `season/winter` — Winter flows
- `season/year_round` — Runnable year-round

### `region/` — Geographic area
- `region/california` — California
- `region/sierra_nevada` — Sierra Nevada mountains
- `region/southern_california` — Southern California
- `region/oregon` — Oregon

**Rule:** Apply multiple region tags for geographic nesting (Kern → `region/california` + `region/sierra_nevada`).

### `flow/` — Flow-specific content (optional)
- `flow/low`, `flow/medium`, `flow/high`, `flow/flood`

**Rule:** Only tag when content is specifically about conditions at a particular flow level.

### `status/` — Data accuracy and currency
- `status/verified` — Field-verified, GPS coordinates confirmed accurate
- `status/needs_verification` — Untested or unconfirmed in the field
- `status/outdated` — Known to need updates (conditions changed, trail rerouted, etc.)
- `status/draft` — Work in progress, not ready for use on the river

**Rule:** Default new/untested data to `status/needs_verification`. Always set a status.

## 5. Content Standards

**Safety accuracy is the highest priority in this vault.** Outdated hazard information can injure or kill paddlers.

- **Difficulty ratings** must reflect current conditions. Ratings can change with flow level, fallen trees, or changed channels. Always note the flow conditions a rating applies to.
- **Hazard waypoints** (`feature/hazard`) — describe portage route, why it is dangerous, and current scouting information.
- **GPS accuracy caveat:** Kern and Rogue canyons have poor satellite reception. All guides must preserve the warning that GPS accuracy degrades in deep canyons.
- **River conditions change seasonally.** Files covering specific flow windows should use `flow/` tags and note the flow range they were surveyed at.
- **Date content.** Waypoint collections should have `last_verified: YYYY-MM-DD` in frontmatter. Unverified data within the last season should be noted; data older than 3 years should be tagged `status/needs_verification` or `status/outdated`.

**Voice and format:**
- Direct and informational — written for paddlers who need to act on information quickly
- Safety warnings use plain language, not jargon
- Metric and imperial measurements are both acceptable; be consistent within a file
- No personal anecdotes or subjective opinions without clearly labeling them as such

## 6. Tagging Heuristics (AI Inference)

When inferring tags from file names and content:

- Files in `Kern/` → `river/kern`. "Lower Kern" → `river/kern/lower`, "North Fork" → `river/kern/north_fork`
- Files in `San Joaquin/` → `river/san_joaquin` + `region/california` + `region/sierra_nevada`
- Files in `Rouge River/` → `river/rogue` + `region/oregon`
- "Rapids" in filename → `feature/rapid`; "Campground" → `feature/campground`; "Access" → `feature/access`; "Parking" → `feature/parking`
- Class IV-V rapids content → always add `difficulty/class_iv_v`; do not omit on technical whitewater
- GPX source files → `type/gpx`; `.json` GeoJSON data → `type/data`; `.md` with leaflet block → `type/river_guide`
- Default to `season/year_round` for Kern mainstem; add flow-specific season tags if mentioned in content
- Default to `status/verified` only for field-confirmed data with `last_verified` date set

## 7. Audit Rules

### Flag as FAIL (high priority — safety)
- Difficulty rating missing on any file with `feature/rapid` — infer from content or flag `status/needs_verification`
- `status/outdated` or missing `last_verified` on rapid/hazard waypoint files — safety risk
- Hazard waypoint (`feature/hazard`) with no portage or safety description in the notes section
- Any content suggesting GPS is 100% reliable in canyons — must preserve canyon accuracy disclaimer

### Flag as FAIL (data quality)
- Missing `type/` tag on any file
- Missing `status/` tag on any file
- Missing `river/` tag on a river guide file
- Flat tags like `#kern`, `#rapid`, `#class-iv` — must be hierarchical
- Nested frontmatter objects (breaks Dataview)

### Flag as WARNING (review needed)
- `status/needs_verification` on content that has been in the vault more than one paddling season without update
- `last_verified` date older than 3 years — suggest field re-verification or downgrade to `status/outdated`
- `feature/rapid` file with no difficulty tag and no note explaining why
- River sections with no `season/` tag

### Do NOT flag
- Auto-generated content inside `<!-- BEGIN AUTO-GENERATED MAP -->` / `<!-- END AUTO-GENERATED MAP -->` blocks — script-managed
- `.json` GeoJSON files with no frontmatter — data files, not notes
- `.gpx` binary/XML files — not markdown
- `_Meta/Tests/` files — plugin behavior test cases, non-navigation content
- Missing `feature/` on access, parking, campground-only collections when covered by a more specific `feature/` tag
